## Wave 3b — Refator modular de `whatsapp-webhook/index.ts`

Objetivo: fatiar o `index.ts` (~3.155 linhas) em módulos coesos, sem mudar comportamento. Cada passo é uma extração isolada, validada pela suíte de testes Deno (`index_test.ts`, 19 testes) antes de seguir para o próximo. Se um passo falhar, ele é revertido antes de prosseguir.

### Estratégia geral

- **Movimentação pura**: cortar funções/constantes do `index.ts` e colá-las em um novo arquivo `lib/<nome>.ts` dentro de `supabase/functions/whatsapp-webhook/`.
- **Imports relativos**: cada módulo importa só o que precisa; `index.ts` re-exporta o necessário para os testes.
- **Sem mudança de assinatura nem de lógica** em nenhum passo.
- **Validação ao fim de cada passo**: rodar `supabase--test_edge_functions` em `whatsapp-webhook`. Verde → próximo passo. Vermelho → reverter o passo e investigar.
- **Sem deploy intermediário**: o deploy automático ocorre uma única vez no final, depois que todos os passos passarem.

### Regras de execução

- Antes de cada passo: `code--view` das funções a mover + dependências (lsp se preciso).
- Edição em duas operações por passo: (a) `code--write` do novo módulo, (b) `code--line_replace` removendo do `index.ts` e adicionando o `import`.
- Após cada passo: `supabase--test_edge_functions { functions: ["whatsapp-webhook"] }`.
- Se qualquer teste falhar: reverter o passo (re-inserir as funções no `index.ts`, deletar o módulo) e parar para reportar.
- Não tocar em RLS, schema, secrets, ou config.toml.

### Detalhes técnicos

- Arquivo único alvo: `supabase/functions/whatsapp-webhook/index.ts`.
- Novos arquivos: `supabase/functions/whatsapp-webhook/lib/{language,text-utils,kb,questions,name-extraction,overrides,twilio,state,gate,prompt,extract}.ts`.
- Teste de regressão atual: `supabase/functions/whatsapp-webhook/index_test.ts` (19 testes — Wave 3a).
- O `// @ts-nocheck` no topo do `index.ts` permanece (erros pré-existentes de tipos do Supabase client). Cada novo módulo será escrito com tipagem correta para evitar propagar `@ts-nocheck`.
- Deploy é automático ao final; nenhuma migração de banco envolvida.

- Não escreve testes novos além dos da Wave 3a — se houver demanda por mais cobertura, vira Wave 3c separada.
